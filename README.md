# official-main-pages

本项目用于归档和发布多个独立的 HTML 页面。这些页面可作为通用资源引用，或用于展示、测试、存档等目的。仓库本身不绑定任何特定域名或网站，仅提供静态页面的集中存放与版本管理。

## 项目简介

`official-main-pages` 是一个以 HTML 页面归档为核心的仓库。所有页面均为独立文件，相互之间无依赖关系。本仓库适合需要将多个静态页面集中管理、版本控制或通过 GitHub Pages 等静态托管服务进行发布的场景。

## 目录结构

```
official-main-pages/
├── pages/           # 存放所有独立 HTML 页面
├── assets/          # 共用的样式、脚本、图片等资源文件（可选）
├── README.md        # 本文件
└── LICENSE          # 项目许可文件（如有）
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名建议使用有意义的英文或数字组合。
- `assets/`：存放多个页面可能共用的资源，如 CSS、JavaScript、图片等。若页面完全自包含，可忽略此目录。

## 页面归档说明

- 每个页面应保持独立，不依赖其他页面或外部资源（除非明确说明）。
- 页面可包含自定义样式、脚本、元信息等，但需确保不包含指向特定域名或诱导性内容。
- 页面内容建议保持通用、中性，适用于多种展示或引用场景。
- 如需引用外部库（如字体、图标），请使用无追踪、无限制的公共 CDN。

## 维护说明

- 欢迎提交 Issue 报告页面问题或提出改进建议。
- 如需新增页面，请在 `pages/` 目录下添加新的 HTML 文件，并确保其内容不违反本仓库的使用原则。
- 拉取请求（Pull Request）应附带清晰的修改描述，说明新增或修改页面的用途。
- 仓库维护者会定期检查页面质量，删除无效或不符合归档要求的文件。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件，即可查看页面效果。
3. 若启用 GitHub Pages，可将仓库设置为静态站点，通过 `https://<你的用户名>.github.io/official-main-pages/pages/<文件名>.html` 访问对应页面。

## 许可

本项目默认采用 MIT 许可证（详见 LICENSE 文件）。每个页面作者可自行决定其内容许可，但建议保持与仓库一致。
